Conception du Robot Gamel Trophy

Projet S1 - IUT de Cachan | Initiation au Prototypage

KiCad (CAO) C (Mbed) Électronique Analogique Robotique suiveur de ligne

Introduction

La Gamel Trophy est une compétition emblématique de l'IUT de Cachan qui oppose les étudiants de première année. Le défi consiste à concevoir un robot autonome, logé dans une véritable gamelle, capable de suivre une ligne noire sur un circuit le plus rapidement possible. Ce projet de semestre 1 a été réalisé en binôme : mon partenaire s'est occupé de la partie motorisation, tandis que je me suis chargé de la partie capteur.

Le robot Gamel terminé, intégrant les cartes électroniques et le châssis mécanique.

1. Réalisation du PCB

1.1 Conception avec KiCad

Ma mission consistait à créer une carte de détection précise. La carte capteur est composée de 5 capteurs infrarouges CNY70. Leur rôle est de détecter la transition entre le blanc et le noir sur la piste pour ajuster la puissance envoyée aux moteurs droit ou gauche selon la courbure du chemin. En plus des capteurs, j'ai dû concevoir l'interface électronique complète pour assurer le dialogue avec le microcontrôleur.

Tracé du PCB sous KiCad : optimisation du placement des capteurs pour une détection fluide.

1.2 Brasage du PCB

Après réception du circuit imprimé, j'ai procédé au brasage des composants. Une étape cruciale a été le test préalable des capteurs CNY70. J'ai identifié certaines unités défaillantes lors des tests de réflexion et j'ai dû les remplacer pour garantir la fiabilité du suivi de ligne lors de la compétition.

Carte finale après brasage, prête à être montée à l'avant du châssis.

2. Programmation et Tests

Avant d'écrire le code définitif, j'ai utilisé un banc d'essai pour moteurs afin de tester leur comportement et de bien comprendre le travail réalisé par mon binôme. Cette phase a été essentielle pour calibrer les vitesses de rotation.

Test des motoréducteurs sur banc d'essai pour valider les commandes de puissance.

Enfin, nous avons assemblé les deux cartes dans la "gamelle". La programmation a été réalisée en C avec le logiciel Mbed. Nous avons élaboré un code fonctionnel traitant les signaux analogiques des capteurs pour commander les moteurs de manière proportionnelle.

Développement du code C sous Mbed pour la gestion de l'asservissement.
